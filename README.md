## 先搞清楚：DDoS攻击是怎么把你打死的

你开了家早餐店，店面只有20平米，最多坐10个人。有一天忽然涌进来1000个人，这些人根本不是来吃早餐的，就是来捣乱堵场子的。店里什么都做不了，只能关门。

DDoS就是这个逻辑。攻击者控制大量"僵尸设备"——被感染的家用路由器、摄像头、普通电脑——同时向你的服务器发送请求，把带宽塞满，把CPU跑满，让真正的用户请求进不来。

常见的攻击类型有几种：

- **流量型攻击（L3/L4）**：最暴力的一种，直接用海量数据包淹你。UDP flood、ICMP flood都属于这类，攻击量随便就是几十到几百Gbps。
- **协议攻击（L4）**：利用TCP握手机制的漏洞，比如SYN flood，让服务器一直在等"握手完成"，资源全被占死。
- **应用层攻击（L7/CC攻击）**：这个更难缠。攻击流量看起来和正常用户请求差不多，专门针对HTTP、HTTPS接口，每秒几万次请求把应用服务器拖垮。

三种攻击，防护思路不一样，这也是选VPS的时候要重点看的地方。

## VPS DDoS防护的几种方式，哪种才是真防御

市面上宣称"带DDoS防护"的VPS，实际上差异极大。搞清楚防护原理，才不会被忽悠。

### 流量清洗（Scrubbing Center）

这是最主流的防护方式。当检测到异常流量涌入，系统把所有数据包导入"清洗中心"，把恶意请求过滤掉，只让正常访问通过。

关键指标是**清洗容量**——能抗住多少Gbps的攻击。10Gbps防护和1Tbps防护，不是一个概念。

### 黑洞路由（Black Hole Routing）

当攻击流量超过运营商能处理的上限，很多服务商会把你的IP"拉黑洞"——所有流量，不管正常还是攻击，全部屏蔽。你的服务器直接从网络上消失。

高防服务的核心价值之一，就是在这一步能撑住，不走黑洞路由，或者能快速切换IP自动恢复。

### 行为分析与特征识别

针对应用层的CC攻击，纯流量清洗效果有限。需要分析请求频率、IP信誉库、访问模式，把"假装正常用户"的恶意请求识别出来。

这部分通常由WAF（Web应用防火墙）或更高级的智能防护引擎处理，价格也更贵。

### 三层防火墙（L3 Firewall）

这是VPS层面的基础防护。在网络层直接过滤恶意IP和异常数据包，防御效果不如专业高防服务，但胜在免费、响应快、不需要配置复杂规则。

## 选VPS DDoS防护：几个实用判断标准

最后整理一些选购时的实用清单，省得踩坑：

**1. 问清楚防护容量上限**
10Gbps和100Gbps是质的区别。很多低价服务商说"自带DDoS防护"但不说防护量级，遇到大流量攻击一样直接黑洞。

**2. 看黑洞政策**
被打进黑洞多久能恢复？能不能自动切换IP？这决定了攻击发生时你的业务停多久。

**3. 确认防护层级**
L3/L4防护和L7防护是两回事。如果你的业务会被CC攻击（应用层），纯粹的网络层防护不够用。

**4. 防护是否免费附带**
很多服务商把DDoS防护当增值服务单独收费，每月多加几十美元。Evoxt的L3防火墙是免费内置的，这个加分项值得考虑。

**5. 节点位置与攻击源的关系**
如果你的攻击来源主要在亚洲，部署在亚太的节点（香港、日本、马来西亚）更容易在上游就把流量过滤掉。Evoxt在这些地区都有布点。
